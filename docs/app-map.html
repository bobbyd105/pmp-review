Logic modules

FilePurposeConsumed by
src/quiz/quizSession.js Pure quiz-session engine (no React): random no-repeat question selection (Fisher-Yates, injectable rng), submit/advance state transitions, score and missed-question selectors, and localStorage save/load/clear with validation that rejects corrupt or stale stored sessions. Storage key: pmp-quiz-session-v1. Quiz, QuizResults; unit-tested directly
src/quiz/quizHistory.js Persistent quiz history (no React): builds one entry per completed quiz (timestamp, score, total, per-domain correct/total, missed question ids), append-only localStorage persistence with corrupt-data degradation to empty, and aggregate stats (total quizzes, overall accuracy, per-domain accuracy from summed tallies). Storage key: pmp-quiz-history-v1 — separate from and never touching the session key. Quiz (append on completion), Dashboard (read + aggregate); unit-tested directly
src/studio/contentValidator.js Pure validation for Content Studio (no React): JSON parse with specific error messages, question/lesson rules mirroring the data-contract tests (required fields, valid ECO domains, ≥2 unique options, correct_answer must match an option, lesson body length/placeholder checks, cross-file id uniqueness in both directions, related_question_ids referential integrity), canonical-field-order snippet serialization. Additive only — never mutates the data it checks against. ContentStudio; unit-tested directly
src/coverage/contentCoverage.js Pure planning-catalog validation and derivation: validates modules, units, ratings/statuses, ECO mappings, PMBOK namespaces, source/lesson references, and duplicates; derives aligned questions, lesson objects, summary totals, and filtered unit lists without mutation. CurriculumCoverage; unit-tested directly against the real catalog
scripts/shuffle-question-options.mjs Seeded, deterministic question-option reorder. Assigns correct-answer positions evenly within each option-count group, deterministically orders distractors, preserves correct_answer as an exact string, and is idempotent for a committed seed. npm run questions:shuffle-options; answerDistribution.test.js
scripts/report-length-bias.mjs Pure answer-length analysis plus report generator: measures Unicode character length, correct/average-distractor ratios, strict-longest rate, worst-first ratio flags, and remediation status without changing content. npm run questions:report-length-bias; lengthBias.test.js

The Claude/Codex batch-orchestration controller, launcher, cleanliness helper, workflow contracts, and their tests were archived to archive/orchestration-infrastructure/ (2026-07-15) as out of scope for current work. They are preserved, not deleted, and are no longer part of the active toolchain.

Data sources

FilePurposeSchemaConsumed by
data/questions.json Question bank data: 384 original PMP-style questions across all 3 ECO domains (People 130, Process 155, Business Environment 99). Phase 1 established breadth; Phase 2 added depth through q082; Phases 3–5 added q083–q155; Phase 6 added q156–q174; Phase 7 added q175–q192; Phase 8 added q193–q218; Phase 9 added q219–q244; Phase 10 added q245–q270; Phase 11 added q271–q296; Phase 12 added q297–q384. People tasks now have 16–17 questions, Process tasks have 15–16, and Business Environment tasks have 12–13. Option order is generated reproducibly with seed pmp-options-v1; all four correct-answer positions contain 96 questions. Answer-length cues were editorially remediated in 2026-07 (18.5% strictly-longest correct, zero over the 1.3 length ratio) and are protected by hard gates (decision_log.md #12). Local JSON per constitution Section 3 — no database, no API. Array of objects: id, eco_domain, eco_task, question, options[], correct_answer (exact string match to one entry in options), explanation QuestionBank, Quiz; test files
localStorage key pmp-quiz-session-v1 In-progress/completed quiz session state (per decision_log.md Decision #3): survives page reload; cleared on Start new quiz Object: questionIds[], currentIndex, results[] of {questionId, selected, correct}. Score is derived, not stored. quizSession.js only (components never touch localStorage directly)
data/lessons.json 26 ECO-aligned lessons, one per task. Schema locked by decision_log.md #7. Each lesson's related_question_ids is regenerated at the end of a content phase from exact eco_domain + eco_task matches; questions.json remains the single source of truth for question content. Array of objects: id, eco_domain, eco_task, title, body (markdown-capable text), optional related_question_ids[] whose entries must exist in data/questions.json (enforced by test) Lessons; lessons.data.test.js
data/concept_lessons.json 62 Authored production concept lessons (c001-c062) — the complete Comprehensive Course in module order from PM Foundations through Agile/Hybrid and AI, learner-facing via the Course view. Each lesson carries 3 authored knowledge checks (position/length-disciplined, enforced by test) and curated related_question_ids links into the bank (empty where no genuine bank coverage exists yet — Scrum, Kanban, PMBOK map, EEF/OPA, AI). Reusable glossary/reference/formula links remain intentionally pending until the reference layer ships. Array of concept records with structured objectives, sections, terms, exam traps, flashcards, mappings, deferred-reference arrays, knowledge_checks[] ({question, options[4], correct_answer, explanation}), related_question_ids[], and source refs. Course; conceptLessons.data.test.js; Course.test.jsx
data/prompts.json Prompt library seed data (Slice 6): 8 curated PMP-study prompts with [bracketed] fill-in slots, for the User to paste into an external AI chat of their choice. Plain local text — the app never executes them or calls any API (constitution Section 2). Standalone: no cross-references to questions or lessons. Array of objects: id, title, description (when to use it), prompt (the copyable text) PromptHelper; prompts.data.test.js
data/content_coverage.json Machine-readable curriculum planning catalog: 13 modules and 59 concept units for the Comprehensive Course. Keeps coverage strength separate from lifecycle and requires ECO, PMBOK, and source mappings. Question counts are deliberately derived, not stored. Object with schema/version enums, modules[], and curriculum_units[] containing id, module, title/type, rating/status, mappings, source refs, and existing lesson ids. CurriculumCoverage; contentCoverage.test.js
data/knowledge_graph.json Planning-only instructional graph for all 59 canonical concepts. Separates broader/narrower hierarchy, acyclic prerequisites, reciprocal related concepts, and glossary/formula/reference relationships while mirroring ECO/PMBOK mappings from the coverage catalog. Object with schema/relationship semantics and concepts[]: id/title, parent/child/prerequisite/related concept IDs, ECO/PMBOK mappings, planned/existing lesson IDs, and shared asset IDs. knowledgeLayer.data.test.js; future knowledge/adaptive loaders. No current runtime UI reads it.
data/learning_objectives.json Planning registry for 59 concept lessons and 118 measurable objectives. Defines prerequisites, competencies, Bloom level, mastery-threshold hypothesis, estimated time, and difficulty without reserving production lesson IDs. Object with enums/approval note and planned_lessons[]; planned IDs use PL-C###, objectives use C###-O#. knowledgeLayer.data.test.js; future lesson, assessment, and adaptive metadata. No current runtime UI reads it.
data/glossary_catalog.json Representative 14-entry glossary schema-validation set with definitions, confusions, exam traps, relationships, and source pointers. Explicitly incomplete and not Approved production content. Object with entries[]: semantic ID, term/definition/guidance, concept/lesson/question/formula/reference IDs, source references. knowledgeLayer.data.test.js; future glossary and remediation views.
data/formula_catalog.json Representative 10-formula schema-validation set covering variables, interpretation, direction guidance, memory tips, mistakes, relationships, and source pointers. Requires technical review before production use. Object with formulas[]: semantic ID, equation, variables, interpretation, comparison guidance, tips/mistakes, lesson/question/glossary/source links. knowledgeLayer.data.test.js; future formula and worked-example views.
data/reference_sheet_catalog.json Complete planned inventory of 12 reusable retrieval assets: process/ITTO maps, organizational/contract/risk/stakeholder/agile/Scrum/leadership/communication comparisons, formula sheet, and responsible-AI checklist. Object with supported asset types, reference_sheets[], and required future review metadata. knowledgeLayer.data.test.js; future reference-sheet views.
docs/content/source_topic_index.json Machine-readable metadata for the 11 local-only sources. Stores stable source IDs, filenames, page counts, topic/role tags, and overlaps; contains no copied source text. Object with sources[]: id, file, pages, primary_topics, curriculum_roles, overlaps. CurriculumCoverage; coverage validation/tests
localStorage key pmp-quiz-history-v1 Append-only history of completed quizzes (Slice 3). One entry per completed quiz; never overwritten, only appended. Independent of the session key. Array of: completedAt (ISO), score, total, domains map of {correct, total} per eco_domain, missedQuestionIds[]. Aggregates are derived, not stored. quizHistory.js only
